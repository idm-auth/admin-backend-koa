# Minimal vs Guardrails - IA Rules

## TRIGGERS AUTOMÁTICOS - PRECEDÊNCIA

### SE tentando resolver "rápido" por causa do minimal
→ **ENTÃO** PARE - minimal não é velocidade, é precisão

### SE considerando "forma fácil" por minimal
→ **ENTÃO** PARE - minimal é fazer certo, não fazer rápido

### SE pulando análise por "ser minimal"
→ **ENTÃO** PARE - análise completa é obrigatória sempre

### SE considerando remover telemetria por "minimal"
→ **ENTÃO** telemetria é código mínimo necessário, não verbosidade

### SE não consigo resolver problema
→ **ENTÃO** NUNCA sugira "remover" como solução

### SE pensando "vamos remover X para resolver Y"
→ **ENTÃO** PARE - isso é solução improvisada, não solução real

### SE não sei como resolver
→ **ENTÃO** admita que não sabe, NUNCA invente soluções

### SE tentando "parecer que sei"
→ **ENTÃO** PARE - honestidade é mais valiosa que fingir conhecimento

## AÇÕES OBRIGATÓRIAS

### Interpretação CORRETA de "minimal"
- **Minimal = fazer certo na primeira vez**
- **Minimal = análise completa + implementação enxuta**

### Interpretação INCORRETA de "minimal"
- **Minimal ≠ fazer rápido**
- **Minimal ≠ pular análise**
- **Minimal ≠ "forma fácil"**
- **Minimal ≠ menos tempo pensando**

## PROCESSO OBRIGATÓRIO

### Antes de implementar QUALQUER solução
1. **LEIA AS RULES** primeiro - sempre consulte .amazonq/rules/
2. **LEIA todos os arquivos** relevantes
3. **ENTENDA a arquitetura** existente
4. **IDENTIFIQUE padrões** do projeto
5. **ENTÃO implemente** com precisão

## PADRÕES DE RECONHECIMENTO

### Solução improvisada detectada quando eu penso:
- "Vou fazer rápido sem ler as rules"
- "Vou pular análise para ser minimal"
- "Vou usar forma fácil em vez de seguir padrões"
- "Vou assumir em vez de verificar nas rules"
- "Vou pular telemetria para ser minimal"
- "Vou simplificar validação para ser rápido"
- "Vou remover X para resolver o problema Y"
- "Se não funciona, vamos tirar"
- "Vou inventar uma solução para parecer que sei"
- "Deve funcionar assim..." (sem certeza real)

### Abordagem correta quando eu digo:
- "Não sei como resolver este problema específico"
- "Admito que meu conhecimento é limitado neste aspecto"
- "Tentei X, Y, Z mas não funcionou"
- "Posso investigar A ou B, qual você prefere?"
- "Este pode ser um problema conhecido sem solução atual"

## REGRA DE OURO

**"Minimal = fazer certo de forma enxuta, não fazer rápido de forma errada"**

**"Análise completa + implementação precisa = verdadeiro minimal"**

**"Tempo pensando nunca é desperdício, é investimento em qualidade"**

**"Admitir ignorância é o primeiro passo para o aprendizado real"**
